## Key Design Decisions

### 1. Separation of Concerns
- **UI Layer**: Pure presentation, minimal logic
- **Hook Layer**: Wallet-specific business logic
- **Store Layer**: Global state management
- **Config Layer**: Static configuration and types

### 2. Extensibility
- Easy to add new chains (just add metadata)
- Easy to add new wallets (just add to wallet modal)
- Type-safe with TypeScript
- Modular architecture

### 3. User Experience
- Unified interface across all chains
- Clear visual feedback
- Automatic wallet detection
- Error handling with user-friendly messages

### 4. Performance
- Lazy loading of wallet adapters
- Efficient state updates
- Minimal re-renders
- Persistent state (localStorage)
